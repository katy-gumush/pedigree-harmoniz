# Testing Strategy

Automated coverage is **Python-only**:

| Layer | Location | Needs running app? |
|-------|----------|---------------------|
| Data / CSV | `tests/test_data_level.py` | No |
| REST API | `tests/test_api.py` | No (FastAPI `TestClient`) |
| UI | `tests/ui/test_pedigree_ui.py` | Yes + Playwright Chromium |

**Databricks:** `databricks/run_unit_tests.py` runs pytest over `tests/`; `tests/conftest.py` sets `collect_ignore = ["ui"]`, so the Playwright directory is **not** collected. Install the app package on the cluster (e.g. `%pip install -e .`) plus `pytest` and `httpx`.

**Local UI runs:** `uv run playwright install chromium`, start `uvicorn`, then `uv run pytest tests/ui`. Override the base URL with `PEDIGREE_UI_BASE_URL` if the app is not on port 8000.

## Data rules (summary)

See `tests/dataset_validation.py` and `test_data_level.py`: required columns, unique IDs, parent references, no self-parent, cycle detection on parent edges, sex-role checks on baseline, height/weight sanity.

## API checks (summary)

`tests/test_api.py` validates JSON shape, 404/400 behavior, pedigree depth ≤ 5, alignment with `build_pedigree` / `build_pedigree_network`, and corrupted fixture behavior (last-row-wins duplicates, skipped missing parents, bounded cycles).

## UI checks (summary)

`tests/ui/test_pedigree_ui.py` maps to **UI Level** cases (home list, search filter, dog card, pedigree SVG, dataset switcher, error page, corrupt datasets). Selectors use `data-testid` attributes in Jinja templates.

## Fixtures

CSV fixtures live under **`fixtures/csv/`** (generated by `scripts/generate_corrupted_datasets.py`). The app registers them in `pedigree_app.datasets` alongside the baseline `Dogs Pedigree.csv`.
